Analise o código deste projeto considerando os seguintes aspectos:

## 1. Arquitetura e Design Patterns
- Identifique padrões de design utilizados (MVC, Repository, Factory, Singleton, etc.)
- Avalie se a separação de responsabilidades está adequada
- Verifique se há violações de princípios SOLID
- Analise a estrutura de pastas - está coerente com o padrão identificado?

## 2. Clean Code
- Nomenclatura de variáveis, funções e classes (clara e significativa?)
- Tamanho e complexidade das funções (alguma muito longa ou com múltiplas responsabilidades?)
- Nível de aninhamento de código (excesso de if/else aninhados?)
- Tratamento de erros adequado (try/catch, promessas rejeitadas tratadas?)
- Comentários desnecessários ou código comentado
- Duplicação de código
- Uso de números mágicos / strings mágicas

## 3. Docker e Deploy (Foco Principal)
- **Compatibilidade de binários nativos**: módulos como bcrypt, sharp, sqlite3 estão sendo compilados para Linux dentro do container ou trazidos do host?
- **.dockerignore**: está ignorando node_modules, .env, arquivos de log e cache?
- **Multi-stage builds**: otimização do tamanho da imagem
- **Ordem das camadas no Dockerfile**: dependências primeiro, código depois (aproveita cache)
- **Variáveis de ambiente**: estão sendo passadas corretamente (ARG vs ENV)?
- **Saúde dos containers**: dependências entre serviços com depends_on e healthcheck
- **Volumes**: persistência de dados configurada corretamente
- **Usuário não-root**: container rodando como root desnecessariamente?

## 4. Problemas Específicos para Detectar
- Incompatibilidade de SO (binários Windows rodando no Linux)
- Dependências globais não declaradas no package.json
- Scripts de build incompletos no Dockerfile
- Portas conflitantes ou não expostas
- Permissões de arquivos dentro do container
- Problemas de CORS ou proxy reverso
- Conexão com banco de dados usando localhost em vez do nome do serviço

## 5. Segurança
- Secrets expostos no código ou Dockerfile
- .env versionado incorretamente
- Dependências com vulnerabilidades conhecidas
- Headers de segurança HTTP ausentes

## 6. Sugestões de Melhoria
- Liste 3-5 melhorias priorizadas, com justificativa
- Indique quais problemas impediriam o deploy imediato no Docker

Retorne o resultado em formato estruturado com:
- ✅ Pontos fortes
- ⚠️ Pontos de atenção (não críticos)
- 🔴 Problemas críticos (impeditivos)
- 💡 Recomendações priorizadas
